### 2026-06-02

#### ✨ 新功能

- **勞健保投保級距管理（`settings.html`）**
  - 設定頁新增「🧾 勞健保級距設定」，加盟主以上可見。
  - 預載 115年度（2026年）完整 75 個級距（6,000 ～ 313,000），涵蓋勞保（12.5%）、健保（5.17%）、勞退（6%）三張表。
  - 支援點擊單一級距手動修改金額；可調整費率後一鍵重新計算全部級距。
  - 資料存於 Firestore `settings/insuranceGrades`，費率調整時不需發版。

#### 🐛 修正

- **登入錯誤訊息明確化**
  - 區分「查無此帳號」與「密碼錯誤」，讓使用者清楚知道問題所在，不再統一顯示模糊提示。

- **手機 Google 登入閃退**
  - 手機瀏覽器不支援 popup，原本 `signInWithPopup` 被降級為 redirect 後，`onAuthStateChanged` 先收到 `null` 導致閃回登入頁。
  - 修正：手機裝置改用 `signInWithRedirect`，頁面載入時優先透過 `getRedirectResult()` 處理回傳，成功後直接進入 app，不再走 null → 顯示登入頁的流程。

- **回首頁後自動登出**
  - `handleGoogleRedirectResult` 的 catch 範圍過廣，任何非預期錯誤（網路、環境不支援等）都會觸發 `showLoginScreen()`，導致已登入的使用者回到首頁時被登出。
  - 修正：catch 僅處理 `not-linked` 錯誤，其餘錯誤改為 `console.warn` 並繼續走正常 `onAuthStateChanged` 流程。

- **個人薪資頁（`my-salary.html`）調店後看不到歷史薪資**
  - 調店後 `currentUser.store` 更新至新門市，但歷史薪資記錄仍存放在舊門市。`my-salary.html` 原本只讀目前門市的薪資，導致調店前的月份顯示空白。
  - 修正：載入時若當月記錄找不到本人資料，自動掃描其他門市尋找歷史薪資並切換讀取，使用者無感。

- **劃休截止日延後一天（週日 → 週一）**
  - `getDeadlineForWeek` 由 `weekMon - 8`（前兩週週日）改為 `weekMon - 7`（前一週週一）。
  - 劃休申請提示橫幅同步顯示「下週一 23:59」截止。

#### ⚙️ 程式碼優化

- **調店 / 職稱異動：`users` 同步防呆**
  - 原本查詢 `users` 時需同時比對 `empName` 與 `store`，若兩個 collection 的 `store` 已不同步，查詢會找不到資料並靜默略過更新。
  - 修正：找不到時改用 `empName` 單獨查作為 fallback，確保異動一定會同步至 `users` 集合。

- **Service Worker 升版（v4 → v5）**
  - 確保 `auth.js` 舊快取自動清除，新版 Google redirect 登入邏輯立即生效。

### 2026-05-29

#### ✨ 新功能

- **職稱延後生效（同店）**
  - 員工管理頁新增「職稱生效日」欄位，可設定未來日期才讓職稱異動生效。
  - 未到生效日前帳號保留原職稱，到日後登入（或重新整理）自動套用，並同步更新 Firestore 的 `employees` 子集合；目前待生效職稱會顯示在帳號欄位旁。
  - 調店與職稱同步：若調店時也設了新職稱，兩者在同一天一起生效。

- **待調入員工顯示**
  - 員工管理頁的員工列表同時撈出 `pendingStore` 指向本門市的帳號，讓店長提前看到即將調入的員工。

- **角色預覽（店長以上）**
  - 首頁「管理功能」新增「角色預覽」，店長可暫時切換為其他角色（工讀、正職、副店長）體驗介面，畫面頂部顯示橘色提示條；離開後自動還原真實身份。

- **薪資頁智慧跳轉**
  - 首頁薪資按鈕改為依權限自動導向：店長以上跳 `salary.html`，一般員工跳 `my-salary.html`（個人薪資頁）。

- **個人薪資頁（`my-salary.html`）完整功能**
  - 進入前需通過二次驗證（密碼或 Google），確保薪資資料安全。
  - 顯示月份薪資明細、考勤、代扣、國假/加班出勤，並附月曆視覺化。

- **人事成本分析大改版（`analytics.html`）**
  - 新增四個分析頁籤：💰 人事成本、⚡ 加班/國假、⚠️ 薪資異常、⏱️ 員工時薪。
  - 薪資異常自動偵測：依可調整的門檻（月薪波動幅度、加班時數上限）標記異常，有異常時自動切換至異常頁並顯示警示條。
  - KPI 卡片：即時顯示應發薪資合計、加班費合計等關鍵數字。
  - 門市 / 職稱多選篩選器，跨店比較更便捷。
  - 匯出下拉選單：一鍵匯出「人事成本表」、「加班/國假表」、「時薪表」三種 CSV。
  - 支援班薪資調整：自動計算跨店支援班對各門市薪資的加減。

#### ⚙️ 程式碼優化

- **`auth.js` 重構**
  - 將登入、Google 登入、待生效調店、同店職稱生效等邏輯統一在 `_loadProfile()` 處理，各頁面不再各自實作。

- **Service Worker 升級（v3 → v4）**
  - 快取版本號更新，確保舊快取自動清除。

### 2026-05-24（第二次更新）

#### 🐛 修正

- **調店功能：帳號門市改為生效日當天才切換**
  - 原本執行調店後帳號的 `store` 欄位會立即更新為新門市，導致員工在生效日前開啟首頁「我的班表」時，顯示新門市的空白班表，看不到舊門市已排定的班次。
  - 修正後：若生效日尚未到達，調店資訊改存入帳號的 `pendingStore` + `transferDate` 欄位，`store` 維持舊門市不動。
  - 員工在生效日當天登入（或重新整理頁面）時，系統自動偵測並完成帳號切換，同步更新 Firestore 並清除暫存欄位，員工無感知。

### 2026-05-24

#### 🐛 修正

- **首頁「我的班表」下週無法切換**
  - 修正 `getNextWeekString()` 週次計算錯誤：原本直接對當天加 7 天後計算週次，在週日時會誤算為下下週（W+2），導致即使已發布下週班表仍顯示「尚未發布」。現改為先對齊本週一再加 7 天，確保任何星期幾呼叫結果一致。

#### ✨ 改進

- **首頁下週班表發布狀態即時同步**
  - 改用 Firestore `onSnapshot` 即時監聽，取代原本一次性 `.get()` 查詢。店長在班表管理頁發布後，員工首頁的「下週」按鈕立即啟用，無需重新整理頁面。
  - 修正初始化競爭條件（Race Condition）：原本頁面載入時存在短暫空窗期，使用者在查詢回應前點擊會誤觸「未發布」提示。

#### ⚙️ 程式碼優化

- **提取共用 Firebase 設定檔（`firebase-init.js`）**
  - 將原本重複在 12 個頁面各自內嵌的 Firebase 設定（API Key、專案 ID 等）統一抽離至獨立檔案，減少約 110 行重複程式碼，往後僅需修改單一位置。

- **Service Worker 升級（v1 → v2）**
  - 快取清單新增 `firebase-init.js` 與 `utils.js`，共用資源改為 Cache First 策略，提升頁面切換速度。
  - 版本更新確保舊快取自動清除。

- **`utils.js` 新增 `requireLogin()` 共用函式**
  - 整合各頁面重複撰寫的登入驗證與跳轉邏輯，供未來新頁面統一呼叫。
