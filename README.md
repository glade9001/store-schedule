#### 🐛 修正

- **已發布班表顯示本週前已調走的員工（`schedule-V2.html`）**
  - `isPublished` 條件原本無條件顯示所有 `status: 調走` 員工，導致早已調走的員工在發布後重新出現。

- **人事成本計算與薪資頁公司負擔不一致（`analytics.html`）**
  - `calcPension` 原用 `(底薪+全勤)×0.06` 公式估算，改為直接讀取薪資記錄中的 `pensionEr`（舊資料無此欄則退回公式）。
  - `calcRealCost` 原為 `calcNet + pension`（含員工代扣扣法），修正為 `calcGross + laborEr + healthEr + pensionEr`，與薪資頁實際人事成本一致。
  - 明細表「公司負擔」欄（原「勞退(6%)」）改為涵蓋 `laborEr + healthEr + pensionEr` 合計。

### 2026-06-03

#### ✨ 新功能

- **更新日誌（`settings.html`）**
  - 設定頁新增「📋 更新日誌」，所有登入使用者皆可查看。
  - 直接 fetch GitHub raw README，解析「更新紀錄」段落後結構化顯示，每次 push 即自動同步，無需另行維護。

- **薪資代扣頁改為投保級距自動帶入（`salary.html`）**
  - 代扣區塊新增「投保級距」下拉選單，分「部分工時」和「月薪制」兩群，選項從 `settings/insuranceGrades` 載入（82 個官方級距）。
  - 選擇級距後自動帶入勞保費、健保費（個人自付），欄位永遠唯讀，只能透過選擇級距帶入。
  - 部分工時健保固定在 29,500 最低級距（$458/$1,428），不隨勞保級距變動。
  - 健保旁加入「已投保／未投保」切換，支援有勞保無健保的情境。
  - 投保級距新增「未投保」選項，勞保、健保、雇主負擔全部清零。
  - 勞保、健保、勞退的雇主負擔以灰色小字顯示於各項目下方，作為人事成本統計基礎。
  - 眷屬由「有/無切換＋手填金額」改為填寫眷屬人數，下方即時顯示「健保費 × N人 = $金額」明細。
  - 新欄位 `insuranceGrade`、`dependentCount`、`laborEr`、`healthEr`、`pensionEr` 存入薪資記錄；新月份自動帶入上月選擇。

- **職稱轉換到期自動套用（`auth.js`、`home.html`）**
  - 原本待轉換職稱需等本人登入才觸發，改為店長以上登入時在背景掃描所有到期的 `pendingRole`，自動更新 `account`、`users`、`stores/employees` 三個 collection。

#### 🐛 修正

- **代扣區塊投保級距未顯示**
  - `renderInsuranceSection` 函式已定義但未被呼叫，`col2` 仍沿用舊的手動輸入欄位渲染。
  - 修正：將 `col2` 代扣欄位改為呼叫 `renderInsuranceSection`，投保級距選單正常顯示。

- **員工看不到設定頁與更新日誌**
  - 設定頁進入時有 `canManager()` 擋關，員工被直接導回首頁，無法查看更新日誌。且首頁設定 sheet 中所有通往 `settings.html` 的入口皆為 `display:none`，員工無從進入。
  - 修正一：移除 `settings.html` 頁面級權限擋關，改為各 section 各自控制顯示條件。員工進入後僅見更新日誌；班別設定需店長以上；假日/勞健保需加盟主以上；系統基礎設定需 admin。
  - 修正二：首頁設定 sheet 新增「📋 更新日誌」入口，對所有登入使用者顯示。

- **密碼錯誤顯示原始 JSON**
  - Firebase SDK 新版將密碼錯誤碼改為 `auth/invalid-login-credentials`，原 codeMap 未涵蓋，導致 fallback 顯示原始 JSON 給使用者。
  - 修正：補上 `auth/invalid-login-credentials` 對應訊息，並將 fallback 從 `e.message` 改為「帳號或密碼錯誤，請再次確認」，避免任何未知錯誤碼外露。

- **薪資發布後加盟主仍可修改代扣項目**
  - `isInsuranceReadonly()` 對 `canApprove()` 永遠回傳 `false`，導致加盟主在已發布狀態仍可修改投保級距等代扣欄位。
  - 修正：改為與 `isReadonly()` 一致，`published` 狀態下所有人均不可修改。

- **調店時 sortKey 複製舊門市值導致碰撞**
  - 調店建立新門市員工記錄時，直接展開 `...fromEmpData` 複製了舊門市的 `sortKey`，若目的地門市已有相同值，排序移動（▲▼）會失效。
  - 修正：改為查詢目的地門市員工總數，以 `toEmpCount.size` 指派新 `sortKey`，排在最後。

- **更新日誌各人顯示版本不一致**
  - `dataset.loaded` 快取旗標使同一 session 內只 fetch 一次，不同時間點開啟 modal 的使用者看到不同版本；加上 GitHub CDN 本身也可能快取舊內容。
  - 修正一：移除快取旗標，每次開啟更新日誌都重新 fetch。
  - 修正二：fetch URL 加上時間戳（`?t=Date.now()`），強制繞過 GitHub CDN 快取，確保每次都拿到最新 README。

### 2026-06-02

#### ✨ 新功能

- **勞健保投保級距管理（`settings.html`）**
  - 設定頁新增「🧾 勞健保級距設定」，加盟主以上可見。
  - 預載 115年度（2026年）完整 75 個級距（6,000 ～ 313,000），涵蓋勞保（12.5%）、健保（5.17%）、勞退（6%）三張表。
  - 支援點擊單一級距手動修改金額；可調整費率後一鍵重新計算全部級距。
  - 資料存於 Firestore `settings/insuranceGrades`，費率調整時不需發版。

#### 🐛 修正

- **登入錯誤訊息明確化**
  - 區分「查無此帳號」與「密碼錯誤」，讓使用者清楚知道問題所在，不再統一顯示模糊提示。

- **手機 Google 登入閃退**
  - 手機瀏覽器不支援 popup，原本 `signInWithPopup` 被降級為 redirect 後，`onAuthStateChanged` 先收到 `null` 導致閃回登入頁。
  - 修正：手機裝置改用 `signInWithRedirect`，頁面載入時優先透過 `getRedirectResult()` 處理回傳，成功後直接進入 app，不再走 null → 顯示登入頁的流程。

- **回首頁後自動登出**
  - `handleGoogleRedirectResult` 的 catch 範圍過廣，任何非預期錯誤（網路、環境不支援等）都會觸發 `showLoginScreen()`，導致已登入的使用者回到首頁時被登出。
  - 修正：catch 僅處理 `not-linked` 錯誤，其餘錯誤改為 `console.warn` 並繼續走正常 `onAuthStateChanged` 流程。

- **個人薪資頁（`my-salary.html`）調店後看不到歷史薪資**
  - 調店後 `currentUser.store` 更新至新門市，但歷史薪資記錄仍存放在舊門市。`my-salary.html` 原本只讀目前門市的薪資，導致調店前的月份顯示空白。
  - 修正：載入時若當月記錄找不到本人資料，自動掃描其他門市尋找歷史薪資並切換讀取，使用者無感。

- **劃休截止日延後一天（週日 → 週一）**
  - `getDeadlineForWeek` 由 `weekMon - 8`（前兩週週日）改為 `weekMon - 7`（前一週週一）。
  - 劃休申請提示橫幅同步顯示「下週一 23:59」截止。

#### ⚙️ 程式碼優化

- **調店 / 職稱異動：`users` 同步防呆**
  - 原本查詢 `users` 時需同時比對 `empName` 與 `store`，若兩個 collection 的 `store` 已不同步，查詢會找不到資料並靜默略過更新。
  - 修正：找不到時改用 `empName` 單獨查作為 fallback，確保異動一定會同步至 `users` 集合。

- **Service Worker 升版（v4 → v5）**
  - 確保 `auth.js` 舊快取自動清除，新版 Google redirect 登入邏輯立即生效。

### 2026-05-24

#### 🐛 修正

- **首頁「我的班表」下週無法切換**
  - 修正 `getNextWeekString()` 週次計算錯誤：原本直接對當天加 7 天後計算週次，在週日時會誤算為下下週（W+2），導致即使已發布下週班表仍顯示「尚未發布」。現改為先對齊本週一再加 7 天，確保任何星期幾呼叫結果一致。

#### ✨ 改進

- **首頁下週班表發布狀態即時同步**
  - 改用 Firestore `onSnapshot` 即時監聽，取代原本一次性 `.get()` 查詢。店長在班表管理頁發布後，員工首頁的「下週」按鈕立即啟用，無需重新整理頁面。
  - 修正初始化競爭條件（Race Condition）：原本頁面載入時存在短暫空窗期，使用者在查詢回應前點擊會誤觸「未發布」提示。

#### ⚙️ 程式碼優化

- **提取共用 Firebase 設定檔（`firebase-init.js`）**
  - 將原本重複在 12 個頁面各自內嵌的 Firebase 設定（API Key、專案 ID 等）統一抽離至獨立檔案，減少約 110 行重複程式碼，往後僅需修改單一位置。

- **Service Worker 升級（v1 → v2）**
  - 快取清單新增 `firebase-init.js` 與 `utils.js`，共用資源改為 Cache First 策略，提升頁面切換速度。
  - 版本更新確保舊快取自動清除。

- **`utils.js` 新增 `requireLogin()` 共用函式**
  - 整合各頁面重複撰寫的登入驗證與跳轉邏輯，供未來新頁面統一呼叫。
