# 莉學商行管理系統

門市排班、薪資、特補休、代辦事項一站式管理系統（PWA）。

---

## 更新紀錄

### 2026-05-24（第二次更新）

#### 🐛 修正

- **調店功能：帳號門市改為生效日當天才切換**
  - 原本執行調店後帳號的 `store` 欄位會立即更新為新門市，導致員工在生效日前開啟首頁「我的班表」時，顯示新門市的空白班表，看不到舊門市已排定的班次。
  - 修正後：若生效日尚未到達，調店資訊改存入帳號的 `pendingStore` + `transferDate` 欄位，`store` 維持舊門市不動。
  - 員工在生效日當天登入（或重新整理頁面）時，系統自動偵測並完成帳號切換，同步更新 Firestore 並清除暫存欄位，員工無感知。

---

### 2026-05-24

#### 🐛 修正

- **首頁「我的班表」下週無法切換**
  - 修正 `getNextWeekString()` 週次計算錯誤：原本直接對當天加 7 天後計算週次，在週日時會誤算為下下週（W+2），導致即使已發布下週班表仍顯示「尚未發布」。現改為先對齊本週一再加 7 天，確保任何星期幾呼叫結果一致。

#### ✨ 改進

- **首頁下週班表發布狀態即時同步**
  - 改用 Firestore `onSnapshot` 即時監聽，取代原本一次性 `.get()` 查詢。店長在班表管理頁發布後，員工首頁的「下週」按鈕立即啟用，無需重新整理頁面。
  - 修正初始化競爭條件（Race Condition）：原本頁面載入時存在短暫空窗期，使用者在查詢回應前點擊會誤觸「未發布」提示。

#### ⚙️ 程式碼優化

- **提取共用 Firebase 設定檔（`firebase-init.js`）**
  - 將原本重複在 12 個頁面各自內嵌的 Firebase 設定（API Key、專案 ID 等）統一抽離至獨立檔案，減少約 110 行重複程式碼，往後僅需修改單一位置。

- **Service Worker 升級（v1 → v2）**
  - 快取清單新增 `firebase-init.js` 與 `utils.js`，共用資源改為 Cache First 策略，提升頁面切換速度。
  - 版本更新確保舊快取自動清除。

- **`utils.js` 新增 `requireLogin()` 共用函式**
  - 整合各頁面重複撰寫的登入驗證與跳轉邏輯，供未來新頁面統一呼叫。
